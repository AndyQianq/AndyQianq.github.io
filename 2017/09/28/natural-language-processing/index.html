自然语言处理

n-gram算法

阅读前建议

  • 读者可以阅读《数学之美》（吴军著），里面的内容浅显易懂，适合对计算机领域有兴趣的人群阅读。

正文部分

  • Natural Language Processing是AI的一个子领域，在NLP（自然语言处理）中有一个很有意思的问题：

  • 此处引用百度百科的例子：

    • “我们把香蕉给猴子，因为它们饿了”
    • “我们把香蕉给猴子，因为它们熟透了”

  • 你一定可以很快的看出“它们”在这两个句子里分别指的是猴子和香蕉。那么问题来了，机器是如何识别“它们”在不同句子里的指代呢？

  • 这也是自然语言处理中常见的问题，即语义的理解。
  • 除此之外，自然语言中还存在很多存在歧义的词语，同音的词语，句法也存在模糊性。随着网络不断深入人们的生活，网络用词，例如近年来的“呵呵”“给力”等词语，都增加了问题的难度，甚至还出现了一些根本不存在的词语。在这些问题的上，很多前辈都做出了不少的努力。（如果想要了解这段历史可以阅读《数学之美》系列八贾里尼克的故事和现代语言处理和系列二十一自然语言处理的教父-马库斯）

统计语言模型

  • 成功利用数学方法解决自然语言处理问题的是贾里尼克，就是他提出了统计语言模型。

  • 那么什么是统计语言模型？

    • 如果用S表示一段特定序列的词语w1，w2……wn组成的句子，那么机器对语言的识别从一定角度来看就是知道S在文本中出现的可能性。
    • 即P（S）=P（w1）P（w2|w1）P（w3|w1w2）……P（wn|w1w2……wn-1）
    • 通过马尔科夫假设，我们假定任意一个词wi的出现该来只和它前面的wi-1有关，那么S出现的概率为：
    • P（S）=P（w1）P（w2|w1）P（w3|w2）……P（wi|wi-1）……
    • （这里的wi-1可以取wi-n即一个词可以由前面的N-1个词来决定）

  • 注释：

  • 马尔科夫假设：

    • t+1时刻系统状态的概率分布只和t时刻的状态有关，与t时刻以前的状态无关；
    • 从t时刻到t+1时刻的状态转移和t的值无关。（有兴趣深入了解可以阅读《信息论与编码》电子科技大学出版第二章第七节：马尔可夫信源）

  • 那么在得到一个足够完善的词料库之后，我们通过统计词频，就可以通过计算得到一个句子的出现的概率，自然概率越高，这个句子就符合该语料下的规则。

  • 这里又提出了一个概念即n-gram语言模型

  • 由刚刚的统计语言模型，我们不妨假设wi的概率由w1，……wi-1来决定，那么设这个特定的一组序列w，……，wi-1成为wi的历史（history）

  • 显然我们意识到一个问题：随着历史基元的增长，不同的历史（路径）会按照指数级增长。对于第i个统计基元，历史基元的个数为i-1，如果共有L个不同的基元，则i基元就有Li-1种不同的历史情况。模型中有Lm个自由参数P（wm|w1……wm-1）.

  • 那么这里举个例子，当L=5000，m=3，自由参数的数目为1250亿。自然这时我们必须想一个解决的方法，这里最直接的想法就是减少历史基元的个数即使用等价类，利用一个映射，将w1w2……wi-1映射到 S（w1w2……wi-1）。自然等价类的数目是远远小于原来不同历史基元的数目的。

  • P（wi|w1……wi-1）=P（wi| S（w1w2……wi-1））

  • 有了这个想法，我们自然就要考虑如果划分等价类。基于这种想法，我们提出了n元文法

    • n=1——》uni-gram 即wi独立于历史
    • n=2——》bi-gram 基于1阶马尔可夫链
    • n=3——》tri-gram 基于2阶马尔可夫链

  • 再举一个简单的例子：

    • 给定句子：John read a book
    • 增加标记：(BOS)John read a book(EOS)
    • Unigram：(BOS), John, read, a, book, (EOS)
    • Bigram: ((BOS)John), (John read), (read a), (a book), (book (EOS))
    • Trigram: ((BOS)John read), (John read a), (read a book), (a book (EOS))

  • 即二元文法的概率为：

  • P(John read a book) =P(John(BOS)×P(read|John)×P(a|read)×P(book|a)×P((EOS)|book)
补充
  • 如果汉字的总数为：N
    • 一元语法：
    • 1）样本空间为 N
    • 2）只选择使用频率最高的汉字
    • 二元语法：
    • 1）样本空间为 N2
    • 2）效果比一元语法明显提高
    • 估计对汉字而言四元语法效果会好一些
    • 智能狂拼、微软拼音输入法基于 n-gram.
  • 那么问题又来了，如何得到n-gram文法？
    • 这里引入两个概念：
    • Training data（训练语料）：用于建立模型，确定模型参数的已知语料
    • Maximum Likelihood Evaluation（最大似然估计）：用相对频率计算概率的方法
      例子
  • 例如，这里给定训练语料：
    • “John read Moby Dick”，
    • “Mary read a different book”,
    • “She read a book by Cher”
  • 那么2元文法求句子的概率为多少？（读者最好自己思考一下）
解答

  • 首先我们对句子进行标识

    • (BOS)John read Moby Dick(EOS)
    • (BOS)Mary read a different book(EOS)
    • (BOS)She read a book by Cher(EOS)

  • 很快我们发现，语料中并没有cher这个单词，那么自然这个句子的概率就是0了。

  • 这其实是NLP中常见的问题，即数据匮乏而引起的零概率问题，如何解决这个问题，即我们要说明的第二个重点。

  • (BOS)John read Moby Dick(EOS)

    • P(Cher read a book) = P(Cher|(BOS)×P(read|Cher)×P(a|read)×P(book|a)× P((EOS)|book)

  • 没有处理：

    • P(Cher|(BOS)) = 0/3
    • P(read|Cher) = 0/1
    • P(a|read) = 2/3
    • P(book|a) = 1/2
    • P((EOS)|book)= 1/2

  • 词汇量：|V|＝11

  • 平滑以后：

    • P(Cher|(BOS)) = (0+1)/(11+3) = 1/14
    • P(read|Cher) = (0+1)/(11+1) = 1/12
    • P(a|read) = (1+2)/(11+3) = 3/14
    • P(book|a) = (1+1)/(11+2) = 2/13
    • P((EOS)|book)= (1+1)/(11+2) = 2/13
    • P(Cher read a book) = 0.00003

  • 同理:

    • P(John|(BOS)) = 2/14, P(read|John) = 2/12,
    • P(a|read) = 3/14, P(book|a) = 2/13,
    • P(John read a book)=P(John|(BOS))×P(read|John)×P(a|read)×P(book|a)×P((EOS)|book)=0.0001

减值法：

  • 基本思想：修改训练样本中事件的实际计数，使样本中(实际出现的)不同事件的概率之和小于1，剩余的概率量分配给未见概率。

总结

  • 其余几种减值法的数据平滑读者可以查询其余资料，这里就不累述了。这里我只是简单的对这四种减值法比较，读者可以通过比较进行针对的查询。

    • Good-Turing 法：对非0事件按公式削减出现的次 数，余留出来的概率均分给0概率事件。
    • Katz 后退法：对非0事件按Good-Turing法计算减 值，余留出来的概率按低阶分布分给0概率事件。
    • 绝对减值法：对非0事件无条件削减某一固定的出现次数值，余留出来的概率均分给0概率事件。
    • 线性减值法：对非0事件根据出现次数按比例削减次数值，余留出来的概率均分给0概率事件。
